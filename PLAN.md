# `dirlir` — antlir2-shaped directory layers from Nix, as hermetic Buck2 toolchains

## Context

antlir2 is Meta's buck2-integrated image builder (analyzed from source at `/tmp/antlir`, plus Vinnie Magro's All Systems Go 2023 talk "antlir2: Deterministic image builds with buck2"). Its key ideas: declarative **features** composed into **layers**; a **depgraph** that validates provides/requires and topo-sorts features before building; a **resolve/materialize split** for package installation (snapshot repos + versionlock JSON → deterministic dnf transaction → install exactly that transaction); isolation via plain `unshare(CLONE_NEWUSER|CLONE_NEWNS)` (deliberately not nspawn, so it runs on RE); local-only btrfs vs RE-able content-addressed directory stacks.

We are building the same concept for **Nix + Buck2**, but the product is a **plain directory tree** (a buck2 tree artifact — natively CAS/RE-compatible), not an image. The directories provide hermetic toolchains (gcc, later ghc) and native deps (glibc via the wrapped gcc, openssl) so that **compile/link actions are fully hermetic and run on RE workers that have neither nix nor /nix/store**.

User decisions (confirmed):
- **Materialization = pure NAR download**: layer-build actions fetch NARs from binary caches by hash and unpack them — no `nix build`, no host-store reads for package content. Nix is needed only at *resolve* time (the snapshot step) and to enter the devshell.
- **RE proof = sandbox hermeticity harness + a live local NativeLink RE backend.**
- **Scope = C toolchain end-to-end first; GHC/haskell_binary as the final milestone (in scope, never blocking).**

Environment facts (verified):
- Host is nix-managed (Lix 2.93.3); **no system python3/gcc/cc on PATH** — everything must be nix-provided, nothing can leak in.
- buck2 in nixpkgs (`unstable-2026-04-15`), and `nixpkgs#buck2.passthru.prelude` exists — a pinned prelude tarball matching that buck2 (solves prelude vendoring/version-skew).
- Unprivileged userns + bind mounts work (`unshare -U -r -m` + `mount --bind` verified) — the shim approach is viable.
- Lix `nix path-info --json` returns the **array** format `[{path, narHash, narSize, references, valid}]`; unrealized paths report `"valid": false` (resolve must build first).

Prior art notes: tweag/buck2.nix wraps `nix build` per package (requires nix on build machines); bazel rules_nixpkgs is RE-incompatible without an NFS-shared store. Our design goes further: after resolve, **no nix at build time at all** — the antlir2 "snapshot repo" is literally cache.nixos.org plus a committed lockfile.

## Concept mapping (antlir2 → dirlir)

| antlir2 | dirlir |
|---|---|
| flavor | pinned nixpkgs (`flake.lock`) |
| snapshot repos + versionlock | cache.nixos.org + committed `nix/lock.json` (store paths, narHashes, narinfo URLs/file hashes) + repo-local `nix/cache/` file cache for custom derivations (the shim) |
| `image.layer` | `dir_layer` rule → plain output-dir tree artifact |
| features (install/rpms/symlink/...) | `feature.nix_packages`, `feature.install`, `feature.ensure_dirs_exist`, `feature.symlink`, `feature.remove` |
| depgraph (Rust+SQLite, plan phase) | `depgraph.py` action → `plan.json` (provides/requires validation + toposort), facts flow parent→child |
| compiler (userns-isolated) | `materialize.py` action: parent copy + NAR fetch/unpack + apply features |
| btrfs (local) / cad-stack (RE) | buck2 tree artifact in CAS (RE-native); parent chaining via `cp -a --reflink=auto` |
| runtime isolation | static userns **shim**: bind-mounts the layer at `/nix/store`, execs the tool |
| packaging (tar/oci/ext4) | not needed — the directory is the product |

Deliberate simplifications: no anon_targets, no BuildPhases (fixed feature-class order + depgraph toposort suffices at prototype scale); no users/groups features.

## Lockfile (`nix/lock.json`) — resolve once, build forever

```json
{ "version": 1, "system": "x86_64-linux",
  "nixpkgs": {"rev": "<from flake.lock>", "narHash": "<from flake.lock>"},
  "caches": ["https://cache.nixos.org", "file://<repo>/nix/cache"],
  "packages": {
    "gcc":     {"attr": "gcc", "storePath": "/nix/store/<h>-gcc-wrapper-...", "closure": ["<sorted, transitively closed, includes root>"]},
    "openssl": {"attr": "openssl", "outputs": {"out": "...", "dev": "...", "bin": "..."}, "closure": ["..."]},
    "nix-store-shim": {"attr": ".#nix-store-shim", "storePath": "...", "closure": ["..."], "cache": "file"}
  },
  "paths": {
    "/nix/store/<h>-glibc-...": {
      "narHash": "sha256-...", "narSize": 123,
      "references": ["..."],
      "nar": {"url": "nar/<filehash>.nar.xz", "fileHash": "sha256:...", "fileSize": 456, "compression": "xz", "cache": 0}
    }
  }
}
```

`nix/lock.bzl` (generated twin, for load-time strings Starlark can't read from JSON):
`NIXPKGS_REV`, `PYTHON3 = "/nix/store/<h>-python3-.../bin/python3"` (the devshell python), `SHIM_STORE_PATH`, `PACKAGES = {name: storePath}`.

**`nix/resolve.py`** (`nix run .#resolve -- gcc binutils openssl python3 hello nix-store-shim`), per attr:
1. `nix build --no-link --json .#legacyPackages.x86_64-linux.<attr>` (flake-pinned ⇒ deterministic store paths).
2. `nix path-info --recursive --json <out>` → closure (parse Lix **array** format; adapter for the CppNix map format).
3. For every closure path, fetch `https://cache.nixos.org/<hashpart>.narinfo`; record URL/FileHash/FileSize/Compression. **Missing narinfo = resolve error** — unless the package is repo-built (the shim), in which case `nix copy --to "file://$REPO/nix/cache?compression=xz"` and reference the file cache. This is the antlir2 guarantee: resolve fails fast so materialize can never miss.
4. Emit `lock.json` + `lock.bzl` (sorted keys, stable formatting). `--check` mode: re-run must be byte-identical (CI guard).

## Starlark design

`dirlir/providers.bzl`:
- `NixLayerInfo(dir: Artifact, facts: Artifact, lock: Artifact, parent)` — `dir` is the product.
- `NixFeatureInfo(feature_json: Artifact, srcs: dict[str, Artifact])`.

`dirlir/features.bzl`: `feature.nix_packages(packages, forest=["bin"])`, `feature.install(src, dst, mode=None)`, `feature.ensure_dirs_exist(path)`, `feature.symlink(link, target)`, `feature.remove(path, must_exist=True)` — load-time records; the `dir_layer` **macro** instantiates each as a `_feature` target (`<layer>--feature-<idx>`, impl = `write_json`). Standalone reusable `feature()` targets also supported.

`dirlir/defs.bzl` — `_dir_layer` rule, attrs: `features` (deps w/ NixFeatureInfo), `parent_layer` (optional dep), `lock` (source, default `root//nix:lock.json`), `_depgraph`/`_nar`/`_materialize` (sources). **Two actions, antlir2's plan/compile split preserved:**

1. **plan** (`dirtree_plan`, `local_only=True`, runs `$PYTHON3 depgraph.py`): validates provides/requires against feature JSONs + parent `facts.json` + lockfile; emits `plan.json` (features in fixed class order — `ensure_dirs_exist` → `nix_packages` → `install`/`symlink` → `remove` — toposorted within by requires-edges; closure list resolved per nix_packages feature). Conflicts/missing deps fail here, cheaply, naming both feature labels.
2. **materialize** (`dirtree_materialize`, `local_only=True`, output via `declare_output("layer", dir=True)`): see below. `local_only` because it needs network + the pinned host-store python; **all downstream consumers are RE-able** — they see only tree artifacts + the static shim (exactly antlir2's local-btrfs / RE-consumers posture).

Provides/requires table (depgraph.py): `install(src,dst)` provides `Entry(dst)`, requires `Dir(parent(dst))`; `symlink` requires target exists (in provides ∪ parent facts); `nix_packages` provides `Dir(nix/store)` + `Entry` per closure path + per forest link; duplicate `Entry` providers = hard error; `remove` = tombstone.

`dirlir/import.bzl`:
- `nix_store_import(name, store_path)` — local_only `cp -a` of a host store path into an artifact. Used ONLY for `nix//:shim` (whose bytes then live in CAS for exec_deps). Falls back to NAR-unpack from the repo file cache if the host store lacks it.
- `dir_subpath(name, layer, path)` — RE-able `cp -rL` (dereference — forest symlinks break when excised) exposing e.g. `include/` or `lib/libssl.so.3` for `prebuilt_cxx_library`.
- `nix_tool(name, layer, path)` — `RunInfo(shim --store <layer.dir> -- <layer.dir>/<path>)`; backs python_bootstrap, command_alias demos, `buck2 run`.

## Materializer (`materialize.py` + `nar.py`, Python 3 stdlib only)

Interpreter = `lock.bzl:PYTHON3` (devshell store path — machines building layers enter `nix develop` for buck2 anyway; this is the minimal bootstrap and the only host-store dependence left).

1. Parent: `cp -a --reflink=auto parent/. out/`; `chmod u+w` **directories** (store trees are r-x; children must be able to add entries; leave dirs writable in output — read-only dirs break `buck2 clean`).
2. `nix_packages`: for each closure path not already present: download `<cache>/<nar.url>` (urllib, retry; shared download cache dir under `buck-out` keyed by fileHash), verify **FileHash** on the compressed bytes, decompress (`lzma` for xz), unpack with `nar.py` (~80-line NAR parser: regular/executable/symlink/directory nodes), verify **NarHash** on the decompressed stream, write to `out/nix/store/<base>`. Then **rewrite every absolute `/nix/store/...` symlink target to relative** (`os.path.relpath` — targets always live inside the tree) — required so the tree artifact is self-contained for CAS; file *contents* (shebangs, PT_INTERP, RPATH) are never touched — the runtime bind mount resolves them. Then the forest: `out/bin/<name> → ../nix/store/<base>/bin/<name>` relative links per requested top dir; collision = error (buildEnv semantics).
3. `install` (from `srcs.json` written `with_inputs=True`), `ensure_dirs_exist`, `symlink`, `remove`: obvious.
4. Walk tree → `facts.json` (`relpath → {type, target, mode}`) for child layers' depgraph.

## The userns shim (`nix/shim/shim.c`, static musl via pkgsStatic, ~180 lines)

`nix-store-shim --store <dir> [--store <dir>...] -- <prog> [args...]`
1. Save cwd; `realpath()` store dirs (pre-unshare).
2. `unshare(CLONE_NEWUSER|CLONE_NEWNS)`; map uid/gid **to self** (`uid_map = "<uid> <uid> 1"`, `setgroups deny`) so outputs keep real ownership.
3. `mount(NULL, "/", NULL, MS_REC|MS_PRIVATE, NULL)`.
4. Fast path (`/nix/store` exists, e.g. this host): bind store dir over `/nix/store` — mounting and masking the host store are the same operation. Multi-store: tmpfs at `/nix/store`, bind each entry of each dir (v1 can require merged layers via `parent_layer` and support a single `--store`).
5. Fallback (no `/nix` — bare RE worker; `mkdir /nix` fails in userns since `/` is owned by unmapped root): build a new root in tmpfs — bind every top-level entry of `/` into it, `mkdir -p nix/store`, bind store dirs, `chroot`, `chdir(saved_cwd)`. (nix-portable's proven trick.)
6. `execvp`. gcc's spawned `cc1`/`as`/`ld` inherit the namespace — no re-wrapping. `prog` is passed as `<layer>/bin/gcc`; its relative forest symlink + the ELF interpreter/shebang absolute paths all resolve through the mount.

Built by `nix build .#nix-store-shim`; resolve pushes it to `nix/cache/` (committed — tiny static binary); imported as `nix//:shim`; toolchains reference it via `attrs.exec_dep(providers=[RunInfo])`.

## C toolchain (`toolchains/nix_cxx.bzl`)

Use nixpkgs **wrapped** `gcc` (cc-wrapper) + `binutils` (bintools-wrapper) — glibc headers/crt/dynamic-linker already baked in, wrapper's bash/coreutils in-closure by construction, shebangs resolve via the mount. (Rejected: `gcc-unwrapped` + manual `-B/--sysroot` — hand-maintaining crt1.o/ld.so paths across nixpkgs bumps.)

```bzl
dir_layer(name = "cxx-toolchain", features = [feature.nix_packages(["gcc", "binutils"], forest = ["bin"])])
```

Model the rule on the vendored `prelude/toolchains/cxx.bzl::_cxx_toolchain_from_cxx_tools_info` (provider set: `CxxToolchainInfo`/`CxxPlatformInfo` with `CCompilerInfo`, `CxxCompilerInfo`, `LinkerInfo(linker_type=LinkerType("gnu"))`, `BinaryUtilitiesInfo`, `CxxInternalTools`). Every tool = `RunInfo(cmd_args(shim, "--store", layer_dir, "--", cmd_args(layer_dir, format="{}/bin/gcc")))` — the layer dir rides along as a cmd_args input ⇒ auto-materialized on RE. `compiler_type="gcc"`; start `DepTrackingMode("none")`, upgrade to `makefile` once python_bootstrap is nix-backed. `toolchains//:python_bootstrap` = prelude `system_python_bootstrap_toolchain` with `interpreter = nix_tool(python layer, "bin/python3")` (RunInfo accepted — verified in prelude source; needed by prelude cxx internal tools like `make_comp_db`/`dep_file_processor`).

**openssl**: `dir_layer(:openssl, nix_packages(["openssl.dev","openssl.out"], forest=["include","lib"]))` + `dir_subpath` targets feeding `prebuilt_cxx_library(header_dirs, shared_lib)`. The demo binary carries `DT_NEEDED libssl.so.3` and runs under the shim with the openssl layer mounted (demonstrates the run-side story; also runs bare on this host).

## Key risks

- **Tree artifacts + symlinks over RE/CAS**: REv2 SymlinkNode handles relative targets; materializer guarantees no absolute symlinks (checked in (c), round-tripped in (h)).
- **zstd-compressed NARs**: cache.nixos.org is xz (stdlib `lzma`); if a cache serves zstd, resolve errors early (or use python 3.14 `compression.zstd` if the devshell python has it).
- **userns blocked on real RE workers**: documented deployment requirement; NativeLink-on-this-host works (verified). Fallback documented.
- **Closure sizes (gcc ~0.4 GB, ghc ~2.5 GB)**: `--reflink=auto` copies; `feature.remove` of `share/{doc,man,info,locale}`; CAS dedupes file-level; keep toolchain layers parent-less.
- **Prelude/buck2 skew**: both from one nixpkgs rev via `passthru.prelude`.
- **Lix vs CppNix JSON shapes**: adapter in resolve.py.
